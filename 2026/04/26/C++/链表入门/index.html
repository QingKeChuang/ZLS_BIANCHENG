C++ 链表入门教程

就像搭积木一样简单！

一、什么是链表？

想象一个寻宝游戏

你参加一个寻宝游戏：

  • 第一张纸条写着：”宝藏不在这一页，下一张纸条在教室门口”
  • 你跑到教室门口，找到第二张纸条，写着：”下一张纸条在操场大树下”
  • 你跑到大树下，找到第三张纸条，写着：”恭喜！宝藏就在这里！”

链表就像这个寻宝游戏：

  • 每张纸条 = 一个”节点”
  • 纸条上的信息 = 存的数据
  • “下一张纸条在哪” = 指向下一个节点的指针

链表 vs 数组

数组 链表
像一排连着的储物柜 像散落在各处的寻宝纸条
每个柜子紧挨着 每张纸条可能在不同地方
找第5个柜子很快 找第5张纸条要从头找
插入新柜子很麻烦（要挪后面的） 插入新纸条很简单（改一下指向就行）

八、链表的优缺点

优点 ✅

  • 插入、删除很快（只要改指向）
  • 不需要提前知道存多少数据
  • 内存可以分散，不用连续空间

缺点 ❌

  • 找第N个元素很慢（要从头数）
  • 每个节点要多存一个指针（占空间）
  • 不能像数组那样随机访问

九、什么时候用链表？

场景 推荐用
要频繁插入、删除 链表 ✅
要频繁查找第N个元素 数组 ✅
不知道要存多少数据 链表 ✅
数据量固定、查找多 数组 ✅
